# Code of Conduct

## Purpose

ArcSub is a public project. Keep collaboration respectful, technically clear, and focused on improving the software.

## Expected Behavior

- be respectful and specific
- assume good intent, but challenge ideas with evidence
- keep feedback focused on code, behavior, docs, and reproducible facts
- help maintain a public issue tracker that is safe for contributors and users

## Unacceptable Behavior

- harassment, threats, or personal attacks
- discriminatory or hateful language
- posting secrets, tokens, private local data, or personal media without permission
- derailing technical discussions with hostility or repeated bad-faith behavior

## Reporting

If you encounter a serious conduct issue, open a private moderation channel if one exists. If that is not available, open a minimal issue asking for a private contact path.

For security-sensitive problems, use [SECURITY.md](./SECURITY.md) instead of a public issue when possible.

## Enforcement

Project maintainers may edit, hide, lock, or remove comments, issues, pull requests, or discussions that violate these expectations. Repeated or severe violations may lead to temporary or permanent removal from the project spaces.
