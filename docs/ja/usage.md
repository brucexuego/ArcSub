# 使い方

## 主な流れ

1. プロジェクトを開く、または作成します。
2. **動画取得** でオンラインリンク、ローカルアップロード、既存プロジェクトファイルから素材を準備します。
3. **音声文字変換** を実行して、文字起こしと字幕を作成します。
4. **高精度翻訳** を実行して、字幕を翻訳します。
5. **動画プレーヤー** で動画と翻訳字幕を一緒に視聴し、ページ上の字幕表示スタイルを調整します。

## モデルを選ぶ

**設定** のモデルカードは優先順に並びます。一番上のカードが、作業ページで最初に選ばれる既定モデルになります。

**音声文字変換** と **高精度翻訳** のページでは、クラウドモデルとローカルモデルを分けて表示します。これにより、選択中のモデルがどこで実行されるかを確認しやすくなります。

- クラウドモデルは、設定したサービスにリクエストを送ります
- ローカルモデルは、必要なモデルファイルをインストールしたあと、自分のPC上で実行されます

片方にモデルがない場合でも、もう片方の利用可能なモデルは選択でき、ない側には空の状態が表示されます。

## 音声文字変換のヒント

- 音声の言語が分かっている場合は、自動検出より直接選ぶほうが安定しやすくなります。
- 字幕のタイミングをより細かく合わせたい場合は、単語時間整列を使います。
- 長い無音部分がある音声や、先に発話区間を見つけたい場合は、音声区間検出を使います。
- 話者ごとに字幕を分けたい場合は、話者分離を使います。pyannote が必要な場合は、先に **設定** でインストールします。

## 翻訳のヒント

- 翻訳前に文字起こしを確認してください。人名、ブランド名、専門用語が多い動画では特に有効です。
- 同じ名前や用語を毎回同じ訳にしたい場合は、用語集を使います。
- 口調、名前の扱い、読みやすさの希望を伝えたい場合は、プロンプト欄を使います。
- 字幕の対応修正を有効にすると、元の字幕1行と翻訳字幕1行の対応を保ちやすくなります。効果は使用するモデルによって変わります。

## ローカルモデル

ローカル ASR またはローカル翻訳を使うには:

1. **設定** を開きます。
2. ローカル ASR またはローカル翻訳モデルの領域を選びます。
3. Hugging Face のモデルIDを入力して確認します。
4. モデルをインストールします。大きなモデルのダウンロードには時間がかかることがあり、バックグラウンドで続行されます。
5. 作業ページに戻り、インストール済みモデルを選びます。

ローカルモデルは自分のPC上で処理しますが、十分なディスク容量と適したハードウェアが必要です。クラウドモデルは始めやすい一方で、処理する音声や文字は設定したサービスに送信されます。

## 動画プレーヤー

**動画プレーヤー** は、動画と翻訳字幕を一緒に視聴するためのページです。動画プレーヤーの字幕スタイル設定は、そのページでの見え方を調整します。
